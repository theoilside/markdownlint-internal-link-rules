## Кастомные правила markdownlint для ссылок

### `CR001` (`encoded-link-spaces`)

Пробелы во всех ссылках должны быть закодированы как `%20`.

- Поддерживает автоисправление через `--fix`.

### `CR002` (`internal-link-leading-slash`)

Локальные ссылки (на Markdown-страницы и вложения) должны начинаться с `/`.

- Поддерживает автоисправление через `--fix` только для ссылок, которые начинаются с:
  - `Внутренняя%20документация`
  - `Агентская%20документация`
  - `Публичная%20документация`
  - `Внутренняя документация`
  - `Агентская документация`
  - `Публичная документация`

### `CR003` (`no-md-file-links`)

Локальные ссылки не должны заканчиваться расширением `.md`.

- Поддерживает автоисправление через `--fix`.

### `CR004` (`internal-file-link-files-directory`)

Локальные ссылки на файлы должны содержать `/_files/` в пути.

Файлом считается ссылка, в последнем сегменте которой есть точка, после которой нет пробела или `%20`.

- Не поддерживает автоисправление через `--fix`.

## Установка в `markdownlint-cli2`

Добавьте правила в свою конфигурацию:

```javascript
module.exports = {
  customRules: [ "./path/to/markdownlint-custom-rule/index.cjs" ],
  config: {
    CR001: true,
    CR002: true,
    CR003: true,
    CR004: true
  }
};
```
